# 目标任务模块分阶段开发计划

目标任务模块采用“**纵向切片**”方式开发。

也就是说，不按“先全部后端、再全部前端”的方式推进，而是每个阶段都完成一个业务模块的闭环：

```text
数据库 / 后端接口 / 前端页面 / 页面验证
```

每个阶段完成后，都要能在页面上跑通并验收，再进入下一阶段。

---

## 第十阶段：第二期统计和仪表盘

### 做什么

第一期先不做复杂统计。  
等目标、任务、进展记录产生真实数据后，再做统计和仪表盘。

第二期可以做：

- 分类维度统计：
    - 每个分类下的目标数量
    - 每个分类下的任务数量
    - 每个分类的完成情况

- 状态维度统计：
    - 进行中目标数量
    - 已完成目标数量
    - 暂停目标数量
    - 放弃目标数量
    - 待处理任务数量
    - 已完成任务数量

- 时间维度统计：
    - 今日完成任务数
    - 本周完成任务数
    - 本月完成任务数
    - 最近 7 天任务完成趋势

- 风险统计：
    - 逾期目标
    - 逾期任务
    - 长时间未推进目标
    - 高优先级未完成任务

### 验收标准

- 可以看到目标任务仪表盘。
- 可以按分类查看目标和任务分布。
- 可以看到本周、本月完成情况。
- 可以看到逾期目标和逾期任务。
- 可以识别长期未推进的目标。
- 统计结果和实际数据一致。

```text
第九阶段：整体联调和第一期验收

第十一阶段：第三期复盘和深度分析
```

## 核心原则

第一期只做主链路：

```text
目标 -> 子目标 -> 任务 -> 完成 -> 进展 -> 进度
```

第二期再做统计仪表盘。

第三期再做复盘分析。

不要一开始就做复杂统计、图表、AI、自动排期。  
先把系统做成**能用**，再做成**好用**，最后再做成**聪明**。
